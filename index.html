🎯 ¿Qué es la Normalización?

La normalización es un proceso sistemático de diseño de bases de datos que organiza los datos para reducir la redundancia y mejorar la integridad de los datos.

🎯 Objetivo Principal

Eliminar la redundancia de datos y asegurar que los datos se almacenen de manera lógica, eficiente y consistente en el tiempo.

👤 4. CLIENTE

Campos: id_cliente, nombre, telefono, direccion, email

🧾 5. FACTURA

Campos: id_factura, id_producto, cantidad, precio_unitario

Campos: id_empleado, nombre, cargo

📂 8. CATEGORIA_PRODUCTO

Campos: id_categoria, nombre, descripcion

Función: Clasificación de tipos de zapatos (deportivos, formales, etc.).

Campos: id_metodoPago, tipo

🔗 Relaciones entre las 9 Tablas

🔗 Mapa Completo de Relaciones

CLIENTE → FACTURA: Un cliente puede realizar múltiples compras (1:N)
Un cliente fiel puede tener muchas facturas a lo largo del tiempo
EMPLEADO → FACTURA: Un empleado puede atender múltiples ventas (1:N)
Cada vendedor registra múltiples ventas durante su jornada
METODO_PAGO → FACTURA: Un método de pago se usa en múltiples facturas (1:N)
Los clientes pueden pagar con efectivo, tarjeta, etc. en diferentes compras
PROVEEDOR ↔ PRODUCTO: Relación muchos a muchos mediante PROVEEDOR_PRODUCTO
Un proveedor puede suministrar varios productos y un producto puede venir de varios proveedores
CATEGORIA_PRODUCTO → PRODUCTO: Una categoría agrupa múltiples productos (1:N)
La categoría "Deportivo" incluye Nike, Adidas, Puma, etc.
FACTURA ↔ PRODUCTO: Relación muchos a muchos mediante FACTURA_PRODUCTO
Una factura puede incluir varios productos y un producto puede estar en varias facturas

📏 Formas Normales Aplicadas

1️⃣ Primera Forma Normal (1NF)

Regla: Cada campo debe contener valores atómicos (indivisibles) y cada registro debe ser único.

✅ En nuestras 9 tablas:

  • Cada talla de zapato es un producto separado
  • No hay campos con múltiples valores
  • Cada registro tiene una clave primaria única

2️⃣ Segunda Forma Normal (2NF)

Regla: Debe estar en 1NF y todos los atributos no clave deben depender completamente de la clave primaria.

✅ En nuestras 9 tablas:

  • Datos del proveedor en tabla separada
  • Información del cliente independiente
  • Eliminadas todas las dependencias parciales

3️⃣ Tercera Forma Normal (3NF)

Regla: Debe estar en 2NF y no debe haber dependencias transitivas.

✅ En nuestras 9 tablas:

  • Categorías en tabla independiente
  • Métodos de pago separados
  • Eliminadas dependencias transitivas

🏆 Resultado de la Normalización

Nuestro sistema cumple con 3NF (Tercera Forma Normal), lo que garantiza:

  • ✅ Cero redundancia de datos
  • ✅ Integridad referencial completa
  • ✅ Flexibilidad para cambios futuros
  • ✅ Optimización del espacio de almacenamiento

📈 Beneficios del Sistema Normalizado

🗃️ Eliminación Total de Redundancia

Cada dato se almacena una sola vez. Las 9 tablas garantizan que no hay duplicación innecesaria.

Ejemplo: Los datos del proveedor se almacenan solo en la tabla proveedor.

🔧 Mantenimiento Centralizado

Cambios en proveedores, categorías o métodos de pago se reflejan automáticamente.

Ejemplo: Cambiar el teléfono de un proveedor actualiza todas las referencias.

🛡️ Integridad Referencial Completa

Las 15+ relaciones con claves foráneas previenen datos inconsistentes.

Ejemplo: No se puede crear una factura sin un cliente válido.

📈 Escalabilidad Máxima

Fácil agregar nuevos elementos sin afectar la estructura existente.

Ejemplo: Agregar nuevas categorías o métodos de pago es simple.

🔍 Análisis de Datos Avanzado

Las relaciones permiten reportes complejos y análisis detallados.

Ejemplo: Reportes de ventas por proveedor, categoría y empleado.

💼 Flexibilidad Comercial Total

Manejo de múltiples escenarios de negocio complejos.

Ejemplo: Un producto puede tener múltiples proveedores.
